# Многоуровневая иерархия серий (Series Hierarchy)

## Обзор

Система теперь поддерживает извлечение и правильное форматирование многоуровневых иерархий серий из имён файлов. Вместо извлечения только последнего уровня, система теперь восстанавливает полную структуру.

## Примеры

### Многоуровневые иерархии (файловый паттерн)

**Файл:**
```
Филимонов Олег - Злой среди чужих (Сид 1. Принцип талиона 1. Геката 1).fb2
```

**Результат:**
- ❌ БЫЛО: `proposed_series="Геката"` (только последний блок)
- ✅ СТАЛО: `proposed_series="Сид\Принцип талиона\Геката"` (полная иерархия)

### Однозровневые серии с номерами томов

**Файл:**
```
Широков Алексей, Шапочкин Александр - Варлок (Варлок 1-3).fb2
```

**Результат:**
- ❌ БЫЛО: `proposed_series="Варлок 1-3"` (с номерами)
- ✅ СТАЛО: `proposed_series="Варлок"` (очищено)

## Архитектура

### 1. BlockLevelPatternMatcher — Сбор блоков

**Файл:** `block_level_pattern_matcher.py`

Класс `BlockLevelPatternMatcher.score_pattern_match()` теперь:
- Токенизирует имя файла на структурные блоки
- Сопоставляет каждый блок с ожидаемым типом (Author, Series, Title)
- **Собирает ВСЕ Series блоки в список** (вместо сохранения только последнего)
- Соединяет блоки с разделителем `. ` (точка-пробел)

**Ключевой фрагмент:**
```python
series_blocks = []  # Collect ALL series blocks

for fname_block, pblock in zip(filename_blocks, pattern_blocks):
    # ... matching logic ...
    if expected_type == "Series":
        series_blocks.append(fname_block.text)  # ADD to list, not replace

# Reconstruct full series hierarchy
series_block = '. '.join(series_blocks) if series_blocks else None
```

**Пример работы:**

Input: `"Сид 1. Принцип талиона 1. Геката 1"`

Токены:
1. `"Сид 1"` → type=Series ✓
2. `"Принцип талиона 1"` → type=Series ✓ 
3. `"Геката 1"` → type=Series ✓

Результат: `"Сид 1. Принцип талиона 1. Геката 1"` (все блоки сохранены!)

### 2. pass2_series_filename.py — Нормализация

**Файл:** `passes/pass2_series_filename.py`

Функция `_extract_series_from_filename()` теперь:

1. ✅ Получает результат от BlockLevelPatternMatcher: `"Сид 1. Принцип талиона 1. Геката 1"`
2. ✅ **ВСЕГДА** применяет `_extract_main_series_from_multi_level()`
3. ✅ Удаляет номера томов из каждого уровня
4. ✅ Объединяет уровни с разделителем `\` (обратный слеш)

**Ключевой фрагмент:**
```python

processed_series = self._extract_main_series_from_multi_level(series_from_block)
if processed_series:
    return processed_series
```

**Метод `_extract_main_series_from_multi_level()`:**

```python
def _extract_main_series_from_multi_level(self, content: str) -> str:
    """
    Пример: "Сид 1. Принцип талиона 1. Геката 1" → "Сид\Принцип талиона\Геката"
    
    Алгоритм:
    1. Разбить по '. ' (точка-пробел)
    2. Для каждого блока: удалить номер в конце (1, 1-3, и т.д.)
    3. Объединить с '\' (обратный слеш)
    """
    
    if '. ' not in content:
        # Однозровневая серия типа "Варлок 1-3"
        # Просто удалить номеры
        return re.sub(r'\s*[\d\-,–]+\s*$', '', content).strip()
    
    # Многозровневая серия
    parts = content.split('. ')
    hierarchy = []
    
    for part in parts:
        # Удалить номеры в конце каждого уровня
        cleaned = re.sub(r'\s*[\d\-,–]+\s*$', '', part).strip()
        if cleaned:
            hierarchy.append(cleaned)
    
    # Объединить с '\' (разделитель иерархии)
    return '\\'.join(hierarchy)
```

**Примеры обработки:**

| Вход | Выход | Тип |
|------|-------|-----|
| `"Сид 1. Принцип талиона 1. Геката 1"` | `"Сид\Принцип талиона\Геката"` | Многоуровневая |
| `"Варлок 1-3"` | `"Варлок"` | Однозровневая |
| `"Солдат удачи"` | `"Солдат удачи"` | Простая серия |

## История разработки

### Проблема (Март 13, 2026)

При использовании BlockLevelPatternMatcher для многоуровневых серий:
- Паттерн: `"Author - Title (Series. SubSeries. SubSubSeries)"`
- Извлекалось: только последний блок "SubSubSeries"
- Нужно: полная иерархия "Series\SubSeries\SubSubSeries"

### Анализ
Обнаружено, что существующий метод `_extract_main_series_from_multi_level()` в Pass2SeriesFilename (commit 90f68ce) уже содержит нужную логику, но:
1. BlockLevelPatternMatcher собирал только последний блок
2. Условная логика проверяла только наличие `. `

### Решение

**Этап 1:** Модифицировать BlockLevelPatternMatcher
- Переключить с одинарной переменной `series_block` на список `series_blocks`
- Объединить все блоки перед возвратом

**Этап 2:** Убедиться в нормализации
- Оставить условный check (). .')
- ... нет, убрать условие! Всегда применять метод

**Этап 3:** Финальная версия
- Всегда применять `_extract_main_series_from_multi_level()` к результату BlockLevelPatternMatcher
