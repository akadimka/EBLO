# Лог Изменений (Changelog)

Все значительные изменения в проекте fb2parser.

## [Неопубликовано]

### Добавлено
- Поддержка многоуровневых иерархий серий через BlockLevelPatternMatcher
- Автоматическое удаление номеров томов из извлеченных серий
- Поддержка разделителей в block-level matching для более точного паттерн-маппинга
- Penalty система для паттернов, которые неправильно классифицируют известные имена авторов

### Изменено
- **BlockLevelPatternMatcher**: теперь собирает ВСЕ блоки серий, а не только последний
- **BlockLevelPatternMatcher**: добавлена информация о разделителях (delimiter) для каждого блока
- **BlockLevelPatternMatcher.score_pattern_match()**: новый алгоритм scoring с учётом разделителей и penalty
- **pass2_series_filename.py**: всегда применяет `_extract_main_series_from_multi_level()` к результатам BlockLevelPatternMatcher
- **Pass2SeriesFilename**: теперь поддерживает передачу male_names/female_names для validation
- **BlockLevelPatternMatcher scoring algorithm**:
  - Основной score: 1.0 за совпадение типа блока
  - Бонус: +0.1 за совпадение разделителя (важно для разрешения ties)
  - Penalty: -0.5 если паттерн ожидает Title на позиции 0, а первый блок содержит известное имя автора

### Критическое исправление: Разрешение проблемы с "Title - Author" паттерном

- **Проблема:** Файл "Акуленко Евгений - Ротмистр" неправильно извлекал "Ротмистр" как автора
- **Анализ:** Три паттерна имели одинаковый score (0.75) и одинаковый type_match_count (1):
  - "Title - Author" → автор=Ротмистр (НЕПРАВИЛЬНО)
  - "Author - Title" → автор=Акуленко Евгений (ПРАВИЛЬНО)
  - "Author. Title" → автор=Акуленко Евгений (ПРАВИЛЬНО, но не подходит по разделителю)
- **Решение реализовано в три этапа:**
  1. Добавлена информация о разделителях в Block и pattern tokenization
  2. Добавлена система бонусов за совпадение разделителей (0.1 point)
  3. Добавлена система penalty для паттернов, которые ожидают Title на позиции 0, когда первый блок содержит известное имя (-0.5 penalty разрешает tie)
- **Результат:** Теперь "Акуленко Евгений - Ротмистр" правильно извлекает "Акуленко Евгений" как автора

### Исправлено
- Филимонов файл теперь показывает "Сид\Принцип талиона\Геката" вместо только "Геката"
- Широков файл теперь показывает "Варлок" вместо "Варлок 1-3"
- Акуленко файл теперь правильно извлекает "Акуленко Евгений" вместо "Ротмистр"

## [2.8] - 2026-03-10

### Критическое исправление: Защита folder_dataset источника

- **Проблема:** Pass2 переписывал `author_source="folder_dataset"` на `"filename"`
- **Решение:** Добавлена защита перед любой перезаписью источника автора
- **Результат:** Файлы в папках авторов теперь правильно сохраняют авторитетный источник

### Поддержка соавторства (Co-authorship)

- Реализована обработка нескольких авторов в одном файле
- Формат CSV: авторы разделены точкой с запятой
- Дополнительная документация: см. [COAUTHORSHIP_FEATURE.md](COAUTHORSHIP_FEATURE.md)
