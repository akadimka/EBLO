## История версий

### v4.9 - Апрель 9, 2026
- 🐛 Исправление: частицы имён (`де`, `ди`, `ван`, `фон`, `ле` и др.) ошибочно капитализировались в Pass3 → `"Фальконес Ильдефонсо Де Сьерра"` вместо `"Фальконес Ильдефонсо де Сьерра"`; добавлен фиксированный набор `_PARTICLES_LOWER`, слова из которого не капитализируются (`passes/pass3_normalize.py`)
- 🐛 Исправление: второй проход обновления коротких форм автора искал авторов только по точному ключу; `"Фальконес Ильдефонсо"` не обновлялся до `"Фальконес Ильдефонсо де Сьерра"`, потому что ключ `"фальконес ильдефонсо"` не был в кэше — только `"фальконес"`; добавлен фолбэк-поиск по первому слову (фамилии) с проверкой что фамилия совпадает (`passes/pass2_filename.py`)
- 🐛 Исправление: файлы с 5-блочной структурой `Author - Series. SubSeries. Книга N. Title` (напр. Клавелл «Азиатская сага 04. Роман о Гонконге. Книга 1. На краю пропасти») не имели подходящего паттерна → block score 0.58 < порога 0.6 → автор брался из метаданных; добавлен паттерн `Author - Series. Series. service_words. Title` (`config.json`)
- 🐛 Исправление: суффикс `(=Альтернативное название)` из метаданных серий не очищался; символ `=` теперь удаляется как часть санитизации недопустимых символов (`passes/pass3_series_normalize.py`)
- 🆕 Санитизация: из итоговых значений `proposed_author` и `proposed_series` удаляются символы, недопустимые в именах папок Windows/Linux: `\ / : * ? " < > | =` (`passes/pass3_normalize.py`, `passes/pass3_series_normalize.py`)
- 🐛 Исправление: regex-паттерн `Author. Series. Title` матчил `Сэнсом К.Дж. - Доминион` → Series=`Дж`, Title=`- Доминион`; добавлена проверка: если захваченный Title начинается с `- ` — паттерн отклоняется как ложное совпадение (`passes/pass2_series_filename.py`)
- 🐛 Исправление: fallback-правило 3 возвращало `Сэнсом К.Дж` как серию, не распознав `К.Дж` как инициальный паттерн автора; добавлена проверка regex `[А-ЯA-Z]\.[А-Яа-я]` — если `potential_series` содержит такой паттерн, он считается именем автора и пропускается (`passes/pass2_series_filename.py`)

### v4.8 - Апрель 8, 2026
- 🐛 Исправление: папки-издатели `"Neoclassic. Crime"`, `"Neoclassic. Расследование"` и т.п. ошибочно распознавались как имена авторов (2 слова → паттерн `(Surname) (Name)`); добавлено `"Neoclassic"` в `filename_blacklist` (`config.json`)
- 🐛 Исправление: суффикс-дизамбигуатор `[Автор]` в конце названия серии (`"Золотой век[Иггульден]"`) теперь удаляется на этапе нормализации (`passes/pass3_series_normalize.py`, шаг 1.7)
- 🐛 Исправление: иностранные/экзотические имена не в словарях (`"Ровена Бергман"`, `"Линдквист Йон Айвиде"`) отвергались `_looks_like_author_name()` → автор ошибочно брался из metadata; добавлено исключение: если все слова ≤3 с заглавной буквы и ни одно не является жанровым ключевым словом → принять как имя собственное (`passes/pass2_filename.py`)
- 🐛 Исправление: при равном score `"Title - Author"` побеждал `"Author - Title"` по порядку в конфиге, что приводило к извлечению названия книги как автора (`"Звёздочка"` вместо `"Линдквист Йон Айвиде"`); паттерн `"Author - Title"` перемещён выше `"Title - Author"` (`config.json`)
- 🐛 Исправление: title guard в `_extract_author_from_filename` не срабатывал, если имя файла содержало `"ё"`, а FB2-метаданные — `"е"` (и наоборот); добавлена нормализация ё→е перед сравнением (`passes/pass2_filename.py`)
- 🐛 Исправление: при расширении однословной фамилии из FB2-metadata (функции `prebuild_author_cache`, `_validate_and_expand_author`, `_try_expand_from_metadata`) возвращалась форма ИФ (`"Хуан Франсиско Феррандис"` вместо `"Феррандис Хуан Франсиско"`); все три точки расширения теперь переставляют совпавшую фамилию на первое место (`passes/pass2_filename.py`)
- 🐛 Исправление: `name_normalizer.py` → `AuthorName._extract_parts()`: для 3+ слов без отчества и без распознанных имён в словаре (иностранные авторы) последнее слово ошибочно считалось фамилией; добавлена эвристика ФИ-конвенции — когда ни первое, ни последнее слово не найдено в словарях имён, первое слово считается фамилией, остальные — именем (`name_normalizer.py`)
- 🐛 Исправление: Pass3 `normalize_format` переставлял слова в filename-авторах в ФИ-порядке даже когда они уже были в нём; добавлена проверка: если после нормализации первое слово изменилось — сохранять исходный ФИ-порядок (`passes/pass3_normalize.py`)
- 🐛 Исправление: гласные в эвристике подсчёта для определения фамилии (`count_vowels`) не включали все кириллические гласные — строка `'aeiouыяеёиoюаAEIOUЫЯЕЁИОЮА'` содержала латинскую `o` вместо кириллической `о`; исправлено (`name_normalizer.py`)

- 🐛 Исправление: полноширокое китайское двоеточие `：` (U+FF1A) в полях автора и серии теперь корректно разделяется — берётся часть после двоеточия (`作者：牛顿不秃顶` → `牛顿不秃顶`); для серий — с защитой от имён-префиксов (`Байши Сюсянь: Название` → `Название`) (`passes/pass3_normalize.py`, `passes/pass3_series_normalize.py`)

- 🐛 Исправление: перечни жанров через запятую (`Путешествия, приключения, фантастика`) теперь распознаются и отклоняются как серии — по шаблону «все части ≤ 2 слова, хотя бы одна в blacklist» (`_is_valid_series`)
- 🐛 Исправление: названия литературных премий (`Литературная премия «Электронная буква – 2019»`) отклоняются как серии — по ключевым словам `премия/award/prize/лауреат/номинант` или году с тире в конце
- 🐛 Исправление: блок «последний шанс» метаданных выполнял только точное сравнение с blacklist; теперь выполняет полный пайплайн валидации (word-boundary blacklist + `_extract_series_from_metadata` + `_remove_blacklist_words` + `_is_valid_series`), как и основной блок
- 🐛 Исправление: порог исключения для blacklist-слов в конце строки повышен с «>0 слов перед ним» до «≥2 слов»; `Современность. Фантастика` теперь отклоняется как жанровая рубрика
- 🐛 Исправление: аббревиатуры вроде `О. Р. З.` больше не образуют иерархию серий — если при разбивке по `". "` появляется однобуквенный компонент, `_extract_main_series_from_multi_level` немедленно возвращает `""` (`passes/pass2_series_filename.py`)

- 🆕 `О. Р. З. или Сказ о том, как Михалыч пить бросал` добавлено в `filename_blacklist` как явная защита (`config.json`)


- 🆕 Новые источники `author_source` / `series_source`:
  - `metadata_folder_confirmed` — метаданные подтверждены совпадением папки
  - `filename_meta_confirmed` — инициалы из имени файла раскрыты через метаданные
- 🐛 Исправление: расширение инициалов «Г.Диксон» → «Гордон Диксон» при совпадении с метаданными (`passes/pass2_filename.py`, метод `_expand_initial_surnames`)

### 🆕 Март 16, 2026 - ИСПРАВЛЕНИЕ: Обработка многоточия как части блока

#### Ellipsis as Block Part, Not Delimiter (Коммит: bc2d5a2)
- **Проблема:** Файлы с многоточием в конце имени (`.`, `..`, `...`, `....`) создавали false series
  - Файл: `"Авраменко Александр - Я не сдаюсь....fb2"` (4 точки!)
  - Паттерн "Author - Series. Title" ошибочно совпадал на `"Author - Title...."`
  - Интерпретация:
    - "Авраменко Александр" → Author ✓
    - "Я не сдаюсь" → **Series** ❌
    - ".." → Title ❌
  - Результат: `proposed_series="Я не сдаюсь"` ← FALSE POSITIVE!

- **Корневая причина:**
  1. **BlockLevelPatternMatcher:** Многоточие трактовалось как разделитель (точка в regex)
  2. **Old patterns:** Позволяли extracted series/title без валидации содержимого

- **Решение (двухчастное):**
  
  **BlockLevelPatternMatcher (block_level_pattern_matcher.py):**
  - Многоточие (`...`, `..`, `....`) рассматривается как часть слова, не разделитель
  - Техника: Placeholder во время tokenization, восстановление после
  ```python
  ELLIPSIS_PLACEHOLDER = "\x00ELLIPSIS\x00"
  text_processed = re.sub(r'\.{2,}', ELLIPSIS_PLACEHOLDER, text)  # Protect
  # ... tokenize using text_processed ...
  block_text = block_text.replace(ELLIPSIS_PLACEHOLDER, "...")  # Restore
  ```
  - Методы: `tokenize_filename()` и `tokenize_pattern()`
  
  **Old Pattern Matching (pass2_series_filename.py):**
  - Отвергнуть matches где Title состоит только из точек
  - Отвергнуть series_candidate если это только точки
  ```python
  if raw_series and all(c == '.' for c in raw_series):
      series_candidate = None  # Not real series
  ```

- **Результат:**
  - ✅ Авраменко "Я не сдаюсь....": `proposed_series=""` (пусто, правильно!)
  - ✅ file_title: `"Я не сдаюсь..."` (правильно идентифицирована)
  - ✅ Нет регрессий: все существующие series сохранены
    - Солдат удачи: 4 файла
    - Цикл Скорпиона: 3 файла
    - Врата Валгаллы: 3 файла (с consensus)

- **Тип изменения:** Bug fix (false-positive series extraction)
