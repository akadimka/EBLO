### 🆕 Март 16, 2026 (Вечер) - УЛУЧШЕНИЕ: Применение паттернов к имёнам папок

#### Series Extraction from Folder Paths with Pattern Matching (Коммит: 002071e)
- **Проблема:** Имена папок серий бралась целиком без очистки авторов и нумерации
  - Папка: `"1941 (Иван Байбаков)"` → `proposed_series="1941 (Иван Байбаков)"` ❌
  - Папка: `"1. Путь в Царьград"` (подсерия) → `proposed_series="1. Путь в Царьград"` ❌
  - Папка: `"2) Варяг"` (подсерия) → `proposed_series="2) Варяг"` ❌
  - Ожидалось: `"1941"` и иерархические `"Ангелы в погонах\Путь в Царьград"`

- **Корневая причина:**
  1. Папки с авторами в скобках: `"Series (Author)"`
  2. Папки подсерий с ведущими номерами: `"1. ", "2. ", "2) "`
  3. Для depth 4+ нужна иерархия, но использовалась только parts[-2]

- **Решение (три части):**

  **1. Применение паттернов из author_series_patterns_in_folders**
  - Используются те же паттерны что и для авторов
  - Паттерны: `"Series (Author)"`, `"Series (Author, Author)"`, `"[Series]"`, `"Series - Description"`, `"Series"`
  - Метод ищет группу "series" в совпадении с паттерном
  - Fallback: удаление авторов regex `r'^(.+?)\s*\([^)]+\)\s*$'`
  
  **2. Удаление ведущих номеров (ДО применения паттернов)**
  - Regex: `r'^\d+[\.\)\-]\s+'` (требует пунктуацию после номера)
  - `"1. Путь в Царьград"` → `"Путь в Царьград"` ✓
  - `"2) Варяг"` → `"Варяг"` ✓
  - `"1941 (Иван)"` → остаётся (нет точки после 1941!)✓
  
  **3. Иерархические серии для depth 4+ (Collection/Series/SubSeries/File)**
  - Depth 3: `parts[-2]` → простая серия
  - Depth 4+: `parts[-3]` + `\` + `parts[-2]` → иерархия
  
  Структура:
  ```
  Серия - «Военная фантастика»\              ← Collection (parts[0])
    Ангелы в погонах (...)\                   ← Main Series (parts[-3])
      1. Путь в Царьград\                     ← Sub-series (parts[-2])
        1. Путь в Царьград.fb2                ← File (parts[-1])
        2. Встречный марш.fb2
        ...
      2. Варяг\                               ← Another Sub-series
        1. Рандеву с «Варягом».fb2
        ...
  ```
  
  Результат depth 4 файлов:
  - `"Ангелы в погонах\Путь в Царьград"` (иерархия сохранена)
  - `"Ангелы в погонах\Варяг"` (ведущий номер удален)

- **Реализация:**
  - `regen_csv.py::_extract_series_from_folder_name()` - основной метод
  - `regen_csv.py::regenerate()` - логика выбора parts[-2] или parts[-3]
  - Загрузка `author_series_patterns_in_folders` в __init__()
  - Импорт `compile_patterns` из pattern_converter

- **Результаты:**
  - ✅ Depth 3 файлы: `"1941"` вместо `"1941 (Иван Байбаков)"`
  - ✅ Depth 3 файлы: `"Адмирал"` вместо `"Адмирал (Михаил Михеев)"`
  - ✅ Depth 4 файлы: `"Ангелы в погонах\Путь в Царьград"` (иерархия)
  - ✅ Depth 4 файлы: `"Ангелы в погонах\Варяг"` (очищено от номера)
  - ✅ Все 672 записи обновлены и проверены
  - ✅ Нет регрессий: простые series остались без изменений

### 🆕 Март 16, 2026 - ИСПРАВЛЕНИЕ: Обработка многоточия как части блока

#### Ellipsis as Block Part, Not Delimiter (Коммит: bc2d5a2)
- **Проблема:** Файлы с многоточием в конце имени (`.`, `..`, `...`, `....`) создавали false series
  - Файл: `"Авраменко Александр - Я не сдаюсь....fb2"` (4 точки!)
  - Паттерн "Author - Series. Title" ошибочно совпадал на `"Author - Title...."`
  - Интерпретация:
    - "Авраменко Александр" → Author ✓
    - "Я не сдаюсь" → **Series** ❌
    - ".." → Title ❌
  - Результат: `proposed_series="Я не сдаюсь"` ← FALSE POSITIVE!

- **Корневая причина:**
  1. **BlockLevelPatternMatcher:** Многоточие трактовалось как разделитель (точка в regex)
  2. **Old patterns:** Позволяли extracted series/title без валидации содержимого

- **Решение (двухчастное):**
  
  **BlockLevelPatternMatcher (block_level_pattern_matcher.py):**
  - Многоточие (`...`, `..`, `....`) рассматривается как часть слова, не разделитель
  - Техника: Placeholder во время tokenization, восстановление после
  ```python
  ELLIPSIS_PLACEHOLDER = "\x00ELLIPSIS\x00"
  text_processed = re.sub(r'\.{2,}', ELLIPSIS_PLACEHOLDER, text)  # Protect
  # ... tokenize using text_processed ...
  block_text = block_text.replace(ELLIPSIS_PLACEHOLDER, "...")  # Restore
  ```
  - Методы: `tokenize_filename()` и `tokenize_pattern()`
  
  **Old Pattern Matching (pass2_series_filename.py):**
  - Отвергнуть matches где Title состоит только из точек
  - Отвергнуть series_candidate если это только точки
  ```python
  if raw_series and all(c == '.' for c in raw_series):
      series_candidate = None  # Not real series
  ```

- **Результат:**
  - ✅ Авраменко "Я не сдаюсь....": `proposed_series=""` (пусто, правильно!)
  - ✅ file_title: `"Я не сдаюсь..."` (правильно идентифицирована)
  - ✅ Нет регрессий: все существующие series сохранены
    - Солдат удачи: 4 файла
    - Цикл Скорпиона: 3 файла
    - Врата Валгаллы: 3 файла (с consensus)

- **Тип изменения:** Bug fix (false-positive series extraction)

### 🆕 Март 13, 2026 - Квартет критических исправлений: Service Words, Filename Cleanup, Consensus Logic, Metadata Validation

#### 1. Metadata Validation for BlockLevelPatternMatcher (Коммит: 5761f69)
- **Проблема:** BlockLevelPatternMatcher ошибочно классифицировал "1-2 книги" как Series блок
  - Файл: `"Васильев Сергей - Император из стали. 1-2 книги.fb2"`
  - Ошибка: `series="1-2 книги"` вместо `"Император из стали"` (из metadata)
- **Решение:** Использовать metadata_series как подтверждение для результатов BlockLevelPatternMatcher
  - Добавлен параметр `metadata_series` в `_extract_series_from_filename()`
  - Если metadata совпадает с BlockLevelPatternMatcher результатом → используем результат ✓
  - Если НЕ совпадает → отвергаем и используем fallback методы ✓
- **Результат:** Васильев "1-2 книги" теперь правильно получает "Император из стали"
- **Теория:** Metadata служит легчайшим валидатором (простая проверка совпадения, без отключения мощного pattern matching)

#### 2. Series Author Consensus with Subset Matching (Коммит: 6e7349a)
- **Проблема:** Авторы внутри серии должны быть идентичны, но консенсус вызывал ложные исправления
  - Врата Валгаллы: файл 3 имел "Ипатова" вместо "Ильин, Ипатова" (неправильно!)
  - Наемник: Капитонов был неправильно изменен на Поселягина (разные авторы!)
- **Предыдущие подходы:** ❌ Все вариант 1-4 либо не исправляли Ипатову, либо изменяли Капитонова
- **Решение (Вариант 5):** Matching-based консенсус использует subset detection
  ```python
  def is_author_subset(current_author, consensus_author):
      # Проверить: содержит ли consensus_author все слова из current_author?
      # "Ипатова" ⊂ "Ильин, Ипатова" → ДА (неполный автор)
      # "Капитонов" ⊄ "Поселягин" → НЕТ (разный автор)
  ```
- **Результат:** 
  - ✅ Врата Валгаллы файл 3: Исправлена на "Ильин Сергей, Ипатова Наталия" с source="filename+series-consensus"
  - ✅ Капитонов: Остался "Капитонов Николай" (защищён)
- **Коммит:** 6e7349a

#### 3. Filename Blacklist Marker Cleanup (Коммит: 30292d3)
- **Проблема:** Файлы с "(СИ)" окончанием не извлекали авторов правильно
  - Файл: `"Путь 2. Школа (СИ).fb2"`
  - Ошибка: "(СИ)" создавал дополнительный блок, нарушая block count для паттернов
  - Результат: 16 файлов Игнатова не извлекали автора
- **Решение:** Удалить blacklist маркеры ДО pattern matching
  ```python
  def _clean_filename_for_extraction(filename: str) -> str:
      # Удалить (СИ), (ЛП) перед pattern matching
      return re.sub(r'\s*\([СЛ]И\)\s*', '', filename)
  ```
- **Результат:** Все 16 файлов Игнатова теперь извлекают автора ✓
- **Коммит:** 30292d3

#### 4. Service Word Validation Fix (Коммит: c1f7877)
- **Проблема:** Серия "Цикл Скорпиона" была отвергнута как serve_word потому что слово "цикл" в service_words
  - Файл: `"Мазур - Цикл Скорпиона"`
  - Ошибка: Substring match ("цикл" IN "цикл скорпиона") → отвергнута
  - Ожидается: Только если весь текст = "цикл" или "цикл" в начале
- **Решение:** Word boundary check вместо substring match
  ```python
  # Отвергаем ТОЛЬКО if:
  # 1. Весь текст = serve_word: "цикл" == "цикл" (нет)
  # 2. Serve_word в НАЧАЛЕ: "цикл скорпиона".startswith("цикл ") (нет)
  # Все остальное = OK ✓
  ```
- **Результат:** Мазур серия "Цикл Скорпиона" теперь правильно извлекается ✓
- **Коммит:** c1f7877

### Изменено
- **BlockLevelPatternMatcher**: теперь собирает ВСЕ блоки серий, а не только последний
- **BlockLevelPatternMatcher**: добавлена информация о разделителях (delimiter) для каждого блока
- **BlockLevelPatternMatcher.score_pattern_match()**: новый алгоритм scoring с учётом разделителей и penalty
- **pass2_series_filename.py**: всегда применяет `_extract_main_series_from_multi_level()` к результатам BlockLevelPatternMatcher
- **Pass2SeriesFilename**: теперь поддерживает передачу male_names/female_names для validation
- **Pass2Filename**: добавлена функция `_clean_filename_for_extraction()` для удаления blacklist маркеров
- **Pass2SeriesFilename._is_valid_series()**: теперь использует word boundary check для serve_words
- **Pass4Consensus**: серия автор консенсус использует subset matching для безопасного исправления
- **BlockLevelPatternMatcher scoring algorithm**:
  - Основной score: 1.0 за совпадение типа блока
  - Бонус: +0.1 за совпадение разделителя (важно для разрешения ties)
  - Penalty: -0.5 если паттерн ожидает Title на позиции 0, а первый блок содержит известное имя автора

### Критическое исправление: Разрешение проблемы с "Title - Author" паттерном

- **Проблема:** Файл "Акуленко Евгений - Ротмистр" неправильно извлекал "Ротмистр" как автора
- **Анализ:** Три паттерна имели одинаковый score (0.75) и одинаковый type_match_count (1):
  - "Title - Author" → автор=Ротмистр (НЕПРАВИЛЬНО)
  - "Author - Title" → автор=Акуленко Евгений (ПРАВИЛЬНО)
  - "Author. Title" → автор=Акуленко Евгений (ПРАВИЛЬНО, но не подходит по разделителю)
- **Решение реализовано в три этапа:**
  1. Добавлена информация о разделителях в Block и pattern tokenization
  2. Добавлена система бонусов за совпадение разделителей (0.1 point)
  3. Добавлена система penalty для паттернов, которые ожидают Title на позиции 0, когда первый блок содержит известное имя (-0.5 penalty разрешает tie)
- **Результат:** Теперь "Акуленко Евгений - Ротмистр" правильно извлекает "Акуленко Евгений" как автора
